# BYOVD Monitor

Лёгкий детектор уязвимых драйверов (BYOVD) для пользовательского режима.
Без драйвера ядра, без цифровой подписи — обычное приложение WPF (.NET Framework 4.8).

Администратор задаёт список папок, приложение в реальном времени и при
периодическом обходе считает SHA-256 каждого файла и сверяет с базой известных
уязвимых драйверов (loldrivers.io). При совпадении загорается красная лампа,
проигрывается звук, запись попадает в журнал с кнопкой «открыть папку».

## Что детектирует

Появление **файла** уязвимого драйвера в наблюдаемой папке (загрузки, временные
каталоги, `System32\drivers`, точки сбора артефактов и т. п.). Это обнаружение
файла на диске по хешу, а не перехват загрузки драйвера в ядро — для перехвата
самого момента загрузки нужен подписанный драйвер ядра, что здесь сознательно
не используется.

## Возможности

- Настраиваемый список наблюдаемых папок (с опцией «вложенные папки»).
- Наблюдатель в реальном времени (`FileSystemWatcher`) плюс обход существующих
  файлов при старте и периодически.
- База хешей из loldrivers.io хранится **локально** (`hashes.json`): при первом
  запуске предлагает загрузить, раз в час проверяет обновление и предлагает обновить.
  По сети ходит только за обновлением (условный запрос по ETag).
- Локальная база на каждую корневую папку (baseline): хеши уже проверенных
  чистых файлов запоминаются, и неизменённые файлы при повторных обходах и
  перезапуске не перепроверяются (не хешируются заново). Вредоносные не
  кэшируются и флагуются всегда.
- Дешёвая отсечка по PE-сигнатуре: новый/изменённый файл сначала проверяется
  на «MZ…PE» (читается 68 байт), и только если это PE-образ (.sys/.exe/.dll/
  .ocx/.efi и т. п.) — считается SHA-256. Текстовые/изображения/архивы
  отбрасываются почти бесплатно.
- Второй слой (необязательно) — Team Cymru MHR: новые файлы (от наблюдателя),
  не найденные в loldrivers, проверяются по DNS (по SHA-1) без ключа.
- Ручная проверка любого файла (кнопка «Проверить файл…») по loldrivers и MHR.
- Индикатор-лампа: серая (нет базы/ожидание), зелёная (чисто), красная
  мигающая (обнаружение).
- Звук уведомления при обнаружении (отключаемый).
- Журнал обнаружений: источник совпадения, открыть папку с выделенным файлом,
  скопировать SHA-256.

## Сборка

    dotnet build BYOVDMonitor.csproj -c Release

Готовый файл: `bin\Release\net48\BYOVDMonitor.exe`.
Можно открыть `BYOVDMonitor.csproj` в Visual Studio и собрать обычным образом.

## Где хранятся данные

`%APPDATA%\BYOVDMonitor\`
- `config.json` — настройки и список папок.
- `hashes.json` — локальная копия базы хешей loldrivers (с ETag для проверки обновлений).
- `baseline\*.json` — локальные базы чистых файлов по каждой корневой папке.

## Источники

- Основной (офлайн, выгрузка списка): `https://www.loldrivers.io/api/drivers.json`
  (≈30 МБ, ~2000 хешей). URL меняется в `config.json` (поле `HashListUrl`).
  loldrivers агрегирует несколько источников, включая блок-лист драйверов Microsoft.
- Дополнительный (поштучно, по требованию): Team Cymru MHR через DNS
  (`<sha1>.malware.hash.cymru.com`), без ключа. Включается галочкой в окне.

## Ограничения

- Хешируются файлы до 64 МБ (настраивается, `MaxFileSizeMb`); драйверы маленькие.
- Сверка по SHA-256 файла: переименование файла обнаруживается, пересборка/правка
  байт драйвера — нет (другой хеш).
- Пропуск неизменённых файлов опирается на размер + время изменения; при ручной
  подмене с сохранением этих атрибутов файл будет считаться неизменённым до
  следующей правки. Для гарантии — «Сканировать сейчас» или ручная проверка файла.
- MHR проверяет файлы поштучно (только новые, от наблюдателя) и шлёт SHA-1 файла
  во внешний DNS — это сетевой запрос к третьей стороне; по умолчанию выключен.
- При обходе больших рекурсивных деревьев первый проход может быть долгим
  (последующие — быстрые за счёт локальной базы папки).
